# MulleObjCUnicodeFoundation Library Documentation for AI
<!-- Keywords: unicode, objc, NSString, NSCharacterSet, categories, version, tests -->

## 1. Introduction & Purpose

- Provides Unicode 3.x.x support for mulle-objc by extending Foundation classes via categories.  No new classes are introduced; behavior is exposed through NSString and NSCharacterSet categories and a small version API.
- Solves: give NSString/NSCharacterSet access to Unicode character classes and helpers that the standard Foundation may not provide.
- Key features: class methods returning canonical character sets (capitalized letters, non-base, decomposable, illegal), version accessors, and inline wrappers for private-character checks.
- Relationship: Component of MulleFoundation; depends on MulleObjCStandardFoundation and mulle-objc-list.

## 2. Key Concepts & Design Philosophy

- Minimal, non-invasive API surface: use Objective-C categories to add behaviour to Foundation classes rather than new types.
- Keeps public headers small and portable by using central import/include headers generated by mulle-sourcetree.
- Tests drive intended usage: many unit tests under test/String/NSMutableCharacterSet exercise character-set construction and membership semantics.
- API is stable and versioned via a compile-time macro and exported accessor functions.

## 3. Core API & Data Structures

This section lists the main public headers and their exported symbols. Use headers as authoritative source of signatures.

### 3.1. [MulleObjCUnicodeFoundation.h]

- Purpose: central public header that exposes version information and small inline helpers that forward to the underlying mulle-utf* routines.
- Key symbols:
  - #define MULLE_OBJC_UNICODE_FOUNDATION_VERSION  ((0UL << 20) | (21 << 8) | 0)
  - static inline unsigned int MulleObjCUnicodeFoundation_get_version_major(void)
  - static inline unsigned int MulleObjCUnicodeFoundation_get_version_minor(void)
  - static inline unsigned int MulleObjCUnicodeFoundation_get_version_patch(void)
  - MULLE_OBJC_UNICODE_FOUNDATION_GLOBAL uint32_t MulleObjCUnicodeFoundation_get_version(void);
  - Inline wrappers for private-character checks:
    - mulle_unicode16_is_privatecharacter(uint16_t c)
    - mulle_unicode32_is_privatecharacter(int32_t c)
    - mulle_unicode_is_privatecharacterplane(unsigned int plane)
- Lifecycle: no stateful objects; functions and macros only.

### 3.2. [NSCharacterSet+MulleObjCUnicode.h]

- Purpose: add class factories for common Unicode character sets.
- Public class methods:
  - + (instancetype) capitalizedLetterCharacterSet;
  - + (instancetype) nonBaseCharacterSet;
  - + (instancetype) decomposableCharacterSet;
  - + (instancetype) illegalCharacterSet;
- Usage: call class methods to obtain autoreleased NSCharacterSet instances and use -characterIsMember: or rangeOfCharacterFromSet:.

### 3.3. [NSMutableCharacterSet+MulleObjCUnicode.h]

- Purpose: category placeholder to augment NSMutableCharacterSet with Unicode-aware mutating operations. Public header declares category; implementation and tests show mutation helpers.
- No explicit public selectors declared in the header; consult tests for expected mutating workflows (add/remove, formUnion, invert).

### 3.4. [NSString+MulleObjCUnicode.h]

- Purpose: category for NSString unicode helpers. Header contains commented hints for standard case helpers and a possible mulleDecapitalizedString but declares no additional selectors in the public header.

### 3.5. [MulleObjCDeps+MulleObjCUnicodeFoundation.h]

- Purpose: expose dependency descriptors for runtime loader integration.
- Public selector:
  - + (struct _mulle_objc_dependency *) dependencies;

## 4. Performance Characteristics

- Character set construction is O(r) where r is number of ranges; membership checks are effectively constant-time for typical representations.
- Case conversions are O(n) in string length and may allocate new strings when mapping expands characters (e.g., German ß).
- Memory vs. speed: character sets are implemented for reuse; prefer shared immutable NSCharacterSet instances for heavy workloads.
- Thread-safety: immutable NSCharacterSet is safe for concurrent reads; NSMutableCharacterSet requires external synchronization for mutations.

## 5. AI Usage Recommendations & Patterns

- Best Practices:
  - Use the provided class factory methods to obtain canonical character sets instead of building sets manually.
  - Query version with MulleObjCUnicodeFoundation_get_version* helpers to detect behavior changes.
  - Prefer immutable NSCharacterSet instances for shared use; avoid mutating shared NSMutableCharacterSet without locking.
- Common Pitfalls:
  - Do not assume presence of non-documented selectors; rely on public headers and tests for usable APIs.
  - Category methods may return autoreleased objects; follow typical Foundation memory rules.
- Idiomatic usage: import the project headers via the project's import.h and use class methods on NSCharacterSet and NSString categories.

## 6. Integration Examples

Examples below are concise and show canonical patterns. Style: 3-space indent, Allman braces.

### Example 1: Using a capitalized-letter character set

```objc
#import "MulleObjCUnicodeFoundation.h"
#import "NSCharacterSet+MulleObjCUnicode.h"

int
main()
{
   unichar           ch;
   NSCharacterSet   *cs;

   ch  = 0x00C5;  // 'Å'
   cs  = [NSCharacterSet capitalizedLetterCharacterSet];

   if( [cs characterIsMember: ch])
   {
      printf("is capitalized\n");
   }
   else
   {
      printf("is not capitalized\n");
   }

   return(0);
}
```

### Example 2: Querying library version

```c
#include "MulleObjCUnicodeFoundation.h"

void
print_version()
{
   unsigned int   major;
   unsigned int   minor;
   unsigned int   patch;

   major = MulleObjCUnicodeFoundation_get_version_major();
   minor = MulleObjCUnicodeFoundation_get_version_minor();
   patch = MulleObjCUnicodeFoundation_get_version_patch();

   printf("MulleObjCUnicodeFoundation version %u.%u.%u\n", major, minor, patch);
}
```

### Example 3: Validate string has no illegal characters

```objc
#import "NSCharacterSet+MulleObjCUnicode.h"

BOOL
stringIsValid( NSString *s)
{
   NSCharacterSet   *illegal;

   illegal = [NSCharacterSet illegalCharacterSet];
   return( [s rangeOfCharacterFromSet: illegal].location == NSNotFound);
}
```
